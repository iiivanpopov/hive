# AGENTS.md

## Project Purpose and Stack

- Bun workspace with two packages: `backend` and `frontend`.
- `backend` is a Bun-targeted Hono API for Hive, with `hono-openapi`, Drizzle ORM on SQLite, Bun Redis-backed token/session repositories, and a WebSocket endpoint.
- `frontend` is a Vite + React 19 app using TanStack Router, TanStack Query, TanStack Form, Tailwind CSS v4, and a generated OpenAPI client.
- Auth is cookie-based, not bearer-token based: backend reads a `session_token` cookie and frontend requests are configured with `credentials: 'include'`.
- Backend contract changes propagate into the frontend through generated files in `frontend/api/` from backend `GET /openapi`.

## Repo Structure

- `backend/src/app/entrypoint.ts`: runtime entrypoint; runs migrations, wires mail/token repos, starts cron jobs, exposes `/openapi`, and starts `Bun.serve`.
- `backend/src/app/create-app.ts`: shared app assembly, middleware, CORS, and health route.
- `backend/src/app/router.ts`: mounts module routers; new API surfaces should be added through modules, not inline here.
- `backend/src/modules/*`: feature modules with `*.router.ts`, `*.service.ts`, and `schema/*.schema.ts` kept together.
- `backend/src/db/tables/**/*.table.ts`: Drizzle table sources; generated migrations live in `backend/drizzle/`.
- `backend/tests/it`: app-level integration tests via `hono/testing` with in-memory SQLite and mocked cache/mail.
- `backend/tests/unit`: isolated unit tests.
- `frontend/src/app/main.tsx`: frontend runtime entrypoint; configures the generated client and router.
- `frontend/src/app/routes`: TanStack file-based routes. `_layout` files define layout routes; folders/files prefixed with `-` are route-local support code, not routes.
- `frontend/src/routeTree.gen.ts`: generated by the TanStack Router Vite plugin.
- `frontend/api/`: generated OpenAPI client and TanStack Query helpers.
- `frontend/src/providers/`, `frontend/src/components/ui/`, and route-local `-components` / `-providers` folders are the main UI composition points.

## Conventions and Boundaries

- Use the existing path aliases instead of long relative imports: frontend `@/*` and `@/api/*`; backend `@/*` and `@/tests/*`.
- Do not hand-edit `frontend/src/routeTree.gen.ts` or anything under `frontend/api/`; regenerate them.
- Keep frontend network code on top of the generated client and generated TanStack Query helpers from `frontend/api/@tanstack/`; avoid ad hoc fetch wrappers.
- Preserve cookie-session behavior. Protected backend routes use `session(...)`, which reads `session_token` and validates the request `User-Agent` before setting `c.get('user')`.
- Keep backend request validation and OpenAPI metadata together in router definitions via `describeRoute(...)` and `validator(...)`, with schemas in each module’s `schema/` folder.
- Backend Hono context types come from `backend/src/lib/factory.ts`; middleware populates `user` and `membership` there.
- Drizzle is sourced from `backend/src/db/tables/**/*.table.ts`; table edits are the source of truth, and migration output belongs in `backend/drizzle/`.
- Route-support folders with a leading `-` are intentionally non-routable. Do not “simplify” them into route files.
- Frontend styling assumes Tailwind v4 from `frontend/src/styles/global.css`; CSS lint only targets `frontend/src/**/*.css`.

## Validation Workflow

- Start with the smallest relevant script for the package you changed, then widen scope only if the change crosses package or contract boundaries.
- Run `bun run lint` after TypeScript/TSX changes; add `bun run lint:css` when editing frontend CSS.
- Run `bun run test:unit` and then `bun run test:it` for backend logic, middleware, schema, or API-contract changes.
- Run repo-root `bun run build:dev` when changes affect both packages, generated router/API artifacts, shared config, or anything pre-commit would exercise.
- After backend OpenAPI changes, regenerate the frontend client in `frontend/` with `bun run api:generate` before validating frontend consumers.
- If route files change, rerun a frontend build/dev command so the TanStack Router plugin refreshes `frontend/src/routeTree.gen.ts`.
- Pre-commit currently runs `bun build:dev`, `bun run test`, and `bun lint-staged`; avoid leaving changes that only pass narrower checks.
